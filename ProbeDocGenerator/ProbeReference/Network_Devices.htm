Network Devices > Apple > Apple AirPort (Extreme)

SNMP - Apple AirPort (Extreme)

This probe monitors the custom MIB in an Apple AirPort Extreme Base Station. This probe can monitor the number of clients using the base station and list each one with its signal strength.

There are two versions of the AirPort Extreme. The "UFO" version does not return complete information to clients using the community string "public". To fully utilize this probe on the "UFO" version, you must set the community string to the AirPort Extreme's password. The "Square" version has a settable SNMP community string. To use this probe on the "Square" version, you must supply the SNMP community string set in the AirPort Extreme.

Filename: com.dartware.snmp.airport.ext

Network Devices > Cisco > Cisco IP SLA Jitter

SNMP - Cisco IP SLA Jitter Probe

This probe extracts jitter test data from a Cisco IP SLA agent that is running on a Cisco router or switch. Typically these jitter tests are used to measure jitter, latency, and packet loss for VoIP and video conferencing applications.

SNMP Index is the value used on the "ip sla monitor" command when configuring the IP SLA agent in the Cisco switch or router (see example below). This value identifies the jitter test, and is the SNMP index used by Intermapper to probe the device. To probe for different instances of jitter tests on a single Cisco switch or router, create separate devices on your Intermapper map each with a different SNMP Index.

Latency Alarm Threshold is the ALARM threshold for latency in milliseconds. If Average Latency value exceeds this threshold the device will enter the Alarm state.

Latency Warning Threshold is the WARNING threshold for latency in milliseconds. If the Average Latency value exceeds this threshold the device will enter the Warning state.

Jitter Alarm Threshold is the ALARM threshold for Jitter. If the Average Jitter value exceeds this threshold the device will go into the Alarm state.

Jitter Warning Threshold is the WARNING threshold for Jitter. If the Average Jitter value exceeds this threshold the device will go into the Warning state.

Packet Loss Alarm Threshold is the ALARM threshold for Packet Loss. If the Percent Packet Loss value exceeds this threshold, the device will enter the Alarm state.

Example

Example IOS commands for configuring an IP SLA jitter test to run on a Cisco router or switch:

ip sla monitor 250

type jitter dest-ipaddr w.w.w.w dest-port 50505 source-ipaddr x.x.x.x num-packets 2000 interval 20

request-data-size 256

owner yyyy

tag zzzz

exit

In the above example "250" is the SNMP index to use. This can be any value as long as it is unique. "w.w.w.w" is the IP address of the remote IP SLA responder. "x.x.x.x" is the local IP address of this IP SLA agent. "yyyy" is any text information identifying the owner of the test (e.g., name of network service provider). "zzzz" is any text information identifying this particular test.

To schedule the IP SLA test to run forever:

ip sla monitor schedule 66 life forever start-time now

To start the IP SLA responder on the remote IP SLA responder:

ip sla monitor responder

In the above IOS commands the jitter test does not specify a codec type, and so ICPIF and MOS scores are not available. If the test is modified to include a codec type then minor revisions will be required to this SNMP probe. Also, some routers and switches may not support the MIB variables for ICPIF and MOS scores - this depends on the IOS train.

More information about configuring Cisco IP SLA is available on the www.cisco.com site.

Filename: com.dartware.snmp.cisco-ip-sla.txt

Network Devices > Cisco > Cisco Old CPU MIB

SNMP - Cisco

This probe monitors the CPU and Memory utilization of a Cisco router.

CPU Busy - Alarm is the ALARM threshold for CPU utilization. If the average percentage of CPU usage over a 1 minute interval exceeds this threshold, the device will enter the Alarm state.

CPU Busy - Warning is the WARNING threshold for CPU utilization. If the average percentage CPU usage over a 1 minute interval exceeds this threshold, the device will enter the Warning state.

Low Memory - Alarm is the ALARM threshold for the amount of free memory remaining (in bytes). If the free memory drops below this threshold, the device will enter the Alarm state.

Low Memory - Warning is the WARNING threshold for the amount of free memory remaining (in bytes). If the free memory drops below this threshold, the device will enter the Warning state.

Filename: com.dartware.snmp.cisco
Version: 1.8

Network Devices > UPS > APC UPS AP961x

For APC UPS devices that have the AP961x Network Management Card.

A. Probed MIB(s)

This probe works best with devices which have implemented the listed MIB(s).

(1 of 2) APC UPS MIB [... enterprises.apc.products.hardware.ups / ... 1.3.6.1.4.1.318.1.1.1]

(2 of 2) APC Environmental Monitoring MIB [... enterprises.apc.products.hardware.environmentalMonitor / ... 1.3.6.1.4.1.318.1.1.10]

B. Displayed Values

UPS: model, firmware, status, (battery: capacity, time remaining, temperature, replacement status), (output: load percent, volts, amps, frequency), (input: volts, voltage range over last minute, frequency, last input failure).

Enviromental Monitor: probe name, number of probes, current temperature & humidity, high & low threshold configurations.

(1 of 3) If unit goes onto battery or goes off-line.

(2 of 3) If battery needs replacement.

(3 of 3) If the UPS' internal temperature/humidity threshold is exceeded (must also be enabled).

D. Warnings

(1 of 1) If unit goes onto "Smart Trim" or "Smart Boost"

Network Devices > UPS > Victron UPS

Victron UPS

This probe monitors some needfull USV parameters.

UPS Battery Status - Alarm is the ALARM threshold for Battery Status.

\i\If the value is 2 USV working normal.

\i\If the value is 1 USV is on bypass, the device will enter the Alarm

state.

UPS Battery Remaining - Warning is the WARNING threshold for the

estimated battery time remaining. If the Battery Remaining is less than

this threshold, the device will enter the Warning state.

UPS Battery low Voltage - Warning is the WARNING threshold for the

min. battery voltage. If the Battery voltage is less than this threshold,

the device will enter the Warning state.

Low Input Voltage line 1 - Alarm is the ALARM threshold for min input

voltage on phase 1. If the input voltage drops below this threshold, the

device will enter the Alarm state.

Low Input Voltage line 2 - Alarm is the ALARM threshold for min input

voltage on phase 2. If the input voltage drops below this threshold, the

device will enter the Alarm state.

Low Input Voltage line 3 - Alarm is the ALARM threshold for min input

voltage on phase 3. If the input voltage drops below this threshold, the

device will enter the Alarm state.

Filename: de.medianet.freinet.ups.victron.txt
